ToolNeuron API

A local, OpenAI-compatible inference server running on your device. Point any OpenAI client at it, add your token, and chat with on-device models — no cloud required.

Open Web Chat → Quickstart

# Overview

ToolNeuron exposes a subset of the OpenAI API surface so existing tooling and SDKs work unchanged. Everything runs on the device hosting the Android app.

⚡ OpenAI-compatible

Drop-in /v1/* routes. Use the official SDKs by overriding the base URL.

🔒 Token-secured

Every /v1/* route requires a bearer token generated in the app.

🧠 Multi-modal

Chat, vision, embeddings, speech, transcription and image generation.

📡 Streaming

Server-sent events for token-by-token chat responses.

# Base URL

The server listens on the device's local address. Find the exact URL on the app's Server screen.

http://<device-ip>:<port>        e.g. http://192.168.1.42:11434
📶

The client device must be on the same network as the host. If you opened this page from the server itself, requests can target the current origin directly.

# Authentication

All /v1/* endpoints require a bearer token. Generate and copy it from the app's Server screen, then send it on every request.

Authorization: Bearer tn_sk_xxxxxxxxxxxxxxxxxxxxxxxxxxxx

Public routes that need no token: /health, /, /index.html, /webui, and /docs. A missing or wrong token on a protected route returns 401.

# Quickstart

Send your first chat completion with curl. Swap in your base URL, token and a model id from /v1/models.

curl http://192.168.1.42:11434/v1/chat/completions \
  -H "Authorization: Bearer $TN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "your-model-id",
    "messages": [{"role": "user", "content": "Hello!"}],
    "stream": true
  }'

Using the OpenAI Python SDK:

from openai import OpenAI

client = OpenAI(
    base_url="http://192.168.1.42:11434/v1",
    api_key="tn_sk_...",            # your ToolNeuron token
)

resp = client.chat.completions.create(
    model="your-model-id",
    messages=[{"role": "user", "content": "Hello!"}],
)
print(resp.choices[0].message.content)

# Models

GET/v1/modelsBearer

Lists models currently exposed by the app. Each entry includes a type describing its role.

{
  "object": "list",
  "data": [
    {
      "id": "your-model-id",
      "object": "model",
      "created": 1718000000,
      "owned_by": "toolneuron",
      "type": "gguf",
      "default": true
    }
  ]
}
FieldDescription
idModel identifier to pass as model.
typeRole: gguf (chat), vlm (vision), embedding, etc.
defaultPresent and true for the suggested default model.

# Chat completions

POST/v1/chat/completionsBearer

Generate a chat response. Set stream: true for token-by-token server-sent events; otherwise a single JSON object is returned. Vision models accept image content parts.

ParameterTypeDescription
model requiredstringA model id from /v1/models.
messages requiredarrayConversation turns with role and content. Content may be a string or an array of text / image_url parts.
stream optionalbooleanStream the reply as SSE chunks. Default false.
max_tokens optionalintegerMaximum tokens to generate.
temperature optionalnumberSampling temperature.
top_p optionalnumberNucleus sampling probability.
presence_penalty optionalnumberPenalize tokens already present.
frequency_penalty optionalnumberPenalize frequent tokens.
stop optionalstring / arrayStop sequence(s).

Request:

POST /v1/chat/completions
{
  "model": "your-model-id",
  "messages": [
    {"role": "system", "content": "You are concise."},
    {"role": "user", "content": "Explain embeddings in one line."}
  ],
  "max_tokens": 256,
  "temperature": 0.7,
  "stream": false
}

Streaming chunks arrive as data: lines, terminated by data: [DONE]:

data: {"choices":[{"delta":{"content":"Embeddings"}}]}
data: {"choices":[{"delta":{"content":" map text to vectors."}}]}
data: [DONE]

# Embeddings

POST/v1/embeddingsBearer

Turn text into vectors with an embedding model. Pass a single string or an array of strings in input.

POST /v1/embeddings
{
  "model": "your-embedding-model",
  "input": ["first text", "second text"]
}

# Audio

POST/v1/audio/speechBearer

Text-to-speech. Returns synthesized audio for the given input text.

POST /v1/audio/speech
{ "model": "your-tts-model", "input": "Hello from ToolNeuron", "voice": "default" }
POST/v1/audio/transcriptionsBearer

Speech-to-text. Send audio as multipart/form-data and receive a transcript.

curl http://192.168.1.42:11434/v1/audio/transcriptions \
  -H "Authorization: Bearer $TN_TOKEN" \
  -F model="your-stt-model" \
  -F file=@clip.wav

# Images

Generate, edit, and upscale images with an image model.

POST/v1/images/generationsBearer

Create an image from a text prompt.

POST /v1/images/generations
{ "model": "your-image-model", "prompt": "a neon city skyline at dusk" }
POST/v1/images/editsBearer

Edit an existing image guided by a prompt (and optional mask).

POST/v1/images/upscaleBearer

Increase the resolution of a source image.

# Errors

The server uses standard HTTP status codes.

StatusMeaning
200Success.
400Malformed request body or missing required fields.
401Missing or invalid bearer token.
404Unknown route.
503Feature not configured (e.g. no model assigned for that role).
