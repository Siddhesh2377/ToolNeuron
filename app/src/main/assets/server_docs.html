Overview

Overview

A local, OpenAI-compatible HTTP API running entirely on your device. No cloud, no telemetry — just the model you loaded in the app, exposed over your LAN.

Base URL

The base URL is whatever the server screen shows in the app. Two URLs are presented:

  • From this device: http://127.0.0.1:<port> — always reachable from the phone's own browser, regardless of Wi-Fi state.
  • From LAN: http://<wifi-ip>:<port> — visible to any device on the same Wi-Fi network when the bind mode is ALL_INTERFACES or WIFI_ONLY.

Compatibility

Endpoints follow OpenAI's wire format, so most clients (the OpenAI Python SDK, OpenWebUI, llama-index, LangChain, etc.) work by pointing them at this base URL with the bearer token from the app.

Endpoints at a glance

GET/health
GET/v1/models
POST/v1/chat/completions
Only /health and the bundled web pages (/, /webui, /docs) are public. Every other route requires a bearer token.

Authentication

All non-public endpoints require a bearer token. The token is generated on the device and never leaves it.

Where to get the token

  1. Open the Tool-Neuron app on the device running the server.
  2. Go to Server screen.
  3. Tap the eye icon on the token card to reveal the token (requires the app to be unlocked).
  4. Tap the copy icon, or rotate to generate a new one.

Sending the token

Include it as a standard Authorization: Bearer <token> header on every request.

curl http://127.0.0.1:11434/v1/models \
  -H "Authorization: Bearer tn_sk_..."

What happens if it's missing

The server replies 401 Unauthorized with body { "error": { "code": 401, "message": "missing or invalid bearer token" } } and a WWW-Authenticate: Bearer header.

After 20 consecutive auth failures from the same client address, the server bans that address for 1 hour and replies 403.

Rotating the token

Tap Rotate in the Server screen. The previous token is immediately invalidated in native memory; subsequent requests with the old token return 401 and count toward the ban threshold.

List models

Returns the catalog of models currently loaded by the server.

GET/v1/models

Request

curl http://127.0.0.1:11434/v1/models \
  -H "Authorization: Bearer tn_sk_..."

Response

{
  "object": "list",
  "data": [
    {
      "id": "qwen2.5-7b-instruct-q4_k_m",
      "object": "model",
      "created": 1714080000,
      "owned_by": "tool-neuron"
    }
  ]
}
The catalog only contains the model the server currently has loaded. Switch models from inside the app before starting the server — dynamic load over the wire is intentionally not exposed.

Chat completions

Send a list of messages, get the assistant's response.

POST/v1/chat/completions

Request body

FieldTypeDescription
modelstringRequired. Must match an id from /v1/models.
messagesarrayRequired. Each item is {role, content} where role is one of system / user / assistant.
streambooleanOptional, default false. When true, response is SSE.
max_tokensintegerOptional. Cap on tokens generated.
temperaturenumberOptional. 0.0–2.0.
top_pnumberOptional. Nucleus sampling.
frequency_penaltynumberOptional.
presence_penaltynumberOptional.

Non-streaming example

curl http://127.0.0.1:11434/v1/chat/completions \
  -H "Authorization: Bearer tn_sk_..." \
  -H "Content-Type: application/json" \
  -d '{
    "model": "qwen2.5-7b-instruct-q4_k_m",
    "messages": [
      { "role": "user", "content": "Hello!" }
    ]
  }'

Response

{
  "id": "chatcmpl-...",
  "object": "chat.completion",
  "created": 1714080000,
  "model": "qwen2.5-7b-instruct-q4_k_m",
  "choices": [{
    "index": 0,
    "message": { "role": "assistant", "content": "Hi!" },
    "finish_reason": "stop"
  }]
}

Streaming (SSE)

Set "stream": true to receive tokens as they're generated.

Wire format

Server-Sent Events. Each chunk arrives as data: {json}\n\n. The stream terminates with the literal line data: [DONE]\n\n.

// First chunk announces the role
data: {"id":"chatcmpl-...","object":"chat.completion.chunk","choices":[{"index":0,"delta":{"role":"assistant"}}]}

// Subsequent chunks carry deltas
data: {"id":"chatcmpl-...","choices":[{"index":0,"delta":{"content":"Hi"}}]}
data: {"id":"chatcmpl-...","choices":[{"index":0,"delta":{"content":"!"}}]}

// Final chunk includes finish_reason and an empty delta
data: {"id":"chatcmpl-...","choices":[{"index":0,"delta":{},"finish_reason":"stop"}]}
data: [DONE]

JavaScript example

const r = await fetch("/v1/chat/completions", {
  method: "POST",
  headers: {
    "Authorization": "Bearer " + token,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({ model, messages, stream: true }),
});
const reader = r.body.getReader();
const dec = new TextDecoder();
let buf = "";
while (true) {
  const { value, done } = await reader.read();
  if (done) break;
  buf += dec.decode(value, { stream: true });
  for (const line of buf.split("\n\n")) {
    if (!line.startsWith("data: ")) continue;
    const json = line.slice(6);
    if (json === "[DONE]") return;
    const chunk = JSON.parse(json);
    onDelta(chunk.choices[0]?.delta?.content ?? "");
  }
}
Streaming chunks intentionally omit usage per OpenAI's spec. Token counts are not reported on the wire.

Health

Cheap public ping. No auth needed, returns immediately.

GET/health
curl http://127.0.0.1:11434/health

Response

{ "status": "ok", "service": "tool-neuron", "version": "0.18.5" }

The bundled web UI uses this as a connection probe — it polls every 30 seconds and shows a green dot when the server is reachable.

Errors

All errors return a JSON envelope with error.code, error.message, and (where applicable) error.type.

Codes

StatusTypeCause
400invalid_request_errorMalformed JSON, missing required fields.
401authentication_errorMissing or invalid Bearer token.
403permission_errorClient banned (20 auth fails / hour).
404invalid_request_errorModel id not in /v1/models.
429rate_limit_errorPer-client token bucket exhausted; retry after the Retry-After header.
499server_errorClient cancelled mid-generation.
500server_errorInference engine error or unexpected exception.
503server_errorInference bridge not attached or refused to start.

Envelope shape

{
  "error": {
    "code": 401,
    "message": "missing or invalid bearer token",
    "type": "authentication_error"
  }
}

Rate limits & bans →

Rate limits & bans

Defaults are conservative for a phone serving a LAN; tune them in app code if you control the build.

Token bucket (per-client IP)

DefaultValue
Bucket capacity30 tokens
Refill rate1.0 tokens / second

Each request takes one token. When the bucket empties the server returns 429 with Retry-After: 1.

Auth-fail ban list

DefaultValue
Threshold20 consecutive auth fails
Ban duration3,600,000 ms (1 hour)

Banned clients receive 403 for the duration. The list is stored in memory only and clears on server restart.

Examples

Drop-in snippets for common clients.

OpenAI Python SDK

from openai import OpenAI

client = OpenAI(
    base_url="http://127.0.0.1:11434/v1",
    api_key="tn_sk_...",
)

resp = client.chat.completions.create(
    model="qwen2.5-7b-instruct-q4_k_m",
    messages=[{"role": "user", "content": "Hello!"}],
)
print(resp.choices[0].message.content)

OpenWebUI

In OpenWebUI's Settings → Connections → OpenAI, set the base URL to http://<phone-lan-ip>:11434/v1 and paste the bearer token. The connected chat dropdown auto-detects the loaded model.

Plain fetch (Node / Bun)

const r = await fetch("http://127.0.0.1:11434/v1/chat/completions", {
  method: "POST",
  headers: {
    "Authorization": "Bearer tn_sk_...",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    model: "qwen2.5-7b-instruct-q4_k_m",
    messages: [{ role: "user", content: "Hi!" }],
  }),
});
const data = await r.json();
console.log(data.choices[0].message.content);

llama-index

from llama_index.llms.openai_like import OpenAILike

llm = OpenAILike(
    api_base="http://127.0.0.1:11434/v1",
    api_key="tn_sk_...",
    model="qwen2.5-7b-instruct-q4_k_m",
)
print(llm.complete("Hi").text)

Security notes

What you can rely on, and what's intentionally not in scope yet.

What's enforced

  • Bearer auth on every non-public route. Constant-time comparison.
  • Per-client rate limit + ban list on auth failures.
  • Token rotation from the app invalidates the previous token in native memory immediately.
  • Token storage on device rides the encrypted HXS vault — same key as the rest of app preferences.
  • App lockdown while server runs — chat-side model load/unload is gated so the server's loaded model isn't yanked mid-request.
  • Static model catalog per server lifetime — dynamic load over the wire is not exposed.

What's not enforced (yet)

  • HTTPS / TLS. Phase 1 is plain HTTP. The server is intended for trusted LANs. If you need TLS, place a reverse proxy in front (nginx / caddy on a workstation on the same LAN), or wait for a TLS build flavor.
  • mDNS / Bonjour discovery. Paste the IP manually for now.
  • Per-user / per-key auth. One token, all-or-nothing.
  • Audit log persistence. The audit ring buffer is in-memory only; restart clears it.
  • Streaming usage metrics. Token counts are not reported on the wire (per OpenAI spec for chat.completion.chunk).
Treat the bearer token like a password. Anyone on the LAN with the token can drive your model.
